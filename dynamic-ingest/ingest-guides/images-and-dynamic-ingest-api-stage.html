    Images et API Dynamic Ingest

    Cette rubrique explique comment capturer ou ingérer des images pour une vidéo.

    Introduction

    Vous pouvez ingérer des images de six types à associer à des vidéos Video Cloud :

    affiche
    L'image de l'affiche est affichée dans le lecteur Brightcove avant que l'utilisateur ne lit la vidéo. Il s'agit généralement d'une image capturée à partir de la vidéo, mais n'a pas besoin de l'être.
    imagettes
    L'image miniature est affichée dans les listes de lecture, les écrans de fin, etc. Il s'agit généralement d'une image capturée à partir de la vidéo, mais n'a pas besoin de l'être.
    portrait
    Image utilisée généralement comme affiche en mode portrait sur les téléphones et autres appareils. Requis pour Brightcove Beacon. Les détails de la spécification sont dans le tableau ci-dessous.
    carré
    Image utilisée généralement comme logo de canal ou dans la grille d'affichage. Requis pour Brightcove Beacon. Les détails de la spécification sont dans le tableau ci-dessous.
    large
    Image utilisée généralement comme affiche en mode paysage sur les téléphones et autres appareils. Requis pour Brightcove Beacon. Les détails de la spécification sont dans le tableau ci-dessous.
    ultra-large
    Image utilisée généralement comme affiche sur des écrans ultra-larges tels que les téléviseurs. Requis pour Brightcove Beacon. Les détails de la spécification sont dans le tableau ci-dessous.

    Spécifications

    Le tableau ci-dessous fournit des spécifications pour les images.

    Tableau des détails de l'image
    Type Format Rapport d'aspect (W/H) Largeur minimale (pixels)
    affiche jpg ou png vidéo de match aucun
    imagettes jpg ou png vidéo de match aucun
    portrait jpg ou png 0,6667 (2x3) 500
    carré jpg ou png 1.0 (1x1) 500
    large jpg ou png 1.7778 (16x9) 2 000
    ultra-large jpg ou png 2.3333 21x9 2 000

    Il existe deux façons différentes d'ajouter des images de base d'affiches et de vignettes à vos vidéos à l'aide de l'API Dynamic Ingest :

    • Capturer des images au milieu de la vidéo pendant le transcodage
    • Ingérez vos propres images, soit en même temps que vous ingérez la vidéo, soit séparément

    Configuration

    La configuration des requêtes Dynamic Ingest est la même, que vous ingérez une vidéo, des images, un fichier WebVTT ou tous ces éléments :

    Notez que tous les noms de fichiers doivent être encodés par URI.
    URL de demande
        https://ingest.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/ingest-requests
    Authentification
    L'authentification nécessite un jeton d'accès passé en tant que Bearer jeton dans un Authorization en-tête : 
        Authorization: Bearer {access_token}

    Note sur S3

    Si vos images sont extraites d'un compartiment S3 protégé, vous devez définir une stratégie de compartiment pour permettre à Video Cloud d'accéder aux fichiers. Pour plus de détails, reportez-vous à Utilisation de l'ingestion dynamique avec S3 .

    Variantes multilingues

    Vous pouvez fournir différentes versions d'images à utiliser pour différentes langues. Ceci est géré par Brightcove Beacon pour les applications Brightcove Beacon. L'implémentation générale dans Brightcove Player nécessite actuellement un développement personnalisé.

    La langue d'une image est spécifiée dans la language propriété détaillée dans la section suivante.

    Ingérer des images

    Voici les détails de l'ajout d'images personnalisées pour votre vidéo.

    Champs d'image dans le corps de la requête

    Les images dans le corps de la requête d'ingest sont spécifiées sous la forme d'un tableau d'objets. Les propriétés de l'objet sont détaillées dans le tableau suivant.

    Propriétés de l'objet Image
    Propriété Description Type Obligatoire Exemple
    url Une URL dans laquelle Video Cloud peut récupérer l'image chaîne https://mysite.com/videos/video123/images/poster.png
    language Code de langue de l'image à partir des sous-balises http://www.iana.org/assignments/language-subtag-registry/language-subtag-registry (par défaut peut être défini pour le compte en contactant le support Brightcove) chaîne
    variant Le type de cette image: poster | thumbnail | portrait | square | wide | ultra-wide chaîne wide
    height nombre 2160
    width nombre 3840 
    {
      "images":[
        {
            "url":"https://apis.support.brightcove.com/assets/images/dynamic-ingest/test-images/poster.jpg",
            "variant": "poster",
            "height": 1080,
            "width": 1920
        },
        {
            "url":"https://apis.support.brightcove.com/assets/images/dynamic-ingest/test-images/thumbnail.jpg",
            "variant": "thumbnail",
            "height": 108,
            "width": 292
        },
        {
            "url":"https://apis.support.brightcove.com/assets/images/dynamic-ingest/test-images/square.jpg",
            "language": "de",
            "variant": "square",
            "height": 570,
            "width": 570
        },
        {
            "url":"https://apis.support.brightcove.com/assets/images/dynamic-ingest/test-images/wide.jpg",
            "language": "fr",
            "variant": "wide",
            "height": 1440,
            "width": 2560
        },
        {
            "url":"https://apis.support.brightcove.com/assets/images/dynamic-ingest/test-images/portrait.jpg",
            "language": "es",

            "height": 1200,
            "width": 800
        },

      ]

    Remarques :

    • Pour les images, seules les URL HTTP/HTTPS non protégées ou les URL S3 signées sont actuellement prises en charge
    • Les images peuvent être incluses avec l'ingestion d'une vidéo master, ou séparément
    • Si vous utilisez le système d'acquisition hérité , et vous omettez le width et height champs, les valeurs par défaut du système seront utilisées (miniature: 160x90; affiche: 960x540)
    • Si vous téléchargez votre propre affiche et images miniatures avec un master vidéo et que le profil d'ingeste que vous utilisez inclut les formats associés d'image, vous devez capture-images définir la valeur false

    Redimensionnement automatique

    Lorsque vous ajoutez des affiches et/ou des miniatures personnalisées via Studio ou l'API Dynamic Ingest, Video Cloud les redimensionnera par défaut pour qu'elles correspondent aux tailles d'image définies dans le profil d'ingest par défaut du compte.

    Notez que le redimensionnement et le rembourrage sont les seules modifications Video Cloud ou Brightcove Beacon. Les images ne sont jamais recadrées.

    Remplacer le redimensionnement automatique

    Remarque : la technique ci-dessous s'applique uniquement aux comptes d'ingestion héritée, et non aux comptes de livraison dynamique.

    Si vous souhaitez remplacer ce comportement et que Video Cloud conserve les dimensions réelles de l'image, procédez comme suit :

    1. Créer un nouveau profil d'ingeste personnalisé pour votre compte (ou modifier un profil personnalisé existant)
    2. Inclure les rendus d'image pour le message et la vignette qui ont exactement celles-ci width et height valeurs: 
          {
      "media_type": "image",
      "format": "jpg",
      "label": "poster",
      "width": 9999,
      "height": 9999
    },
    {
      "media_type": "image",
      "format": "jpg",
      "label": "thumbnail",
      "width": 9999,
      "height": 9999
    }
    3. Faites de ce profil le profil par défaut du compte si vous souhaitez toujours enregistrer les images avec leur taille d'origine.

    Maintenant, lorsque vous ajoutez des images personnalisées via Studio ou Dynamic Ingest, elles auront les dimensions d'origine des images source.

    Capturer des images

    Pendant le transcodage, Video Cloud peut capturer un instantané de la vidéo au milieu et l'enregistrer sous forme d'affiche et d'images miniatures.

    Le nombre d'affiches et de vignettes créées par l'Ingest dynamique est un pour chacun. Vous ne pouvez pas créer plusieurs formats associés d'affiche et de vignette même si vous avez plusieurs formats associés d'image dans le profil.

    Pour les profils d'acquisition personnalisés, la capture d'image nécessite l'ajout de deux formats associés spéciaux au profil d'acquisition que vous utilisez. Voir Images dans les profils d'ingest ci-dessous.

    Vous devez également définir l' capture-images indicateur dans les données de votre requête sur true si vous souhaitez que l'affiche et la miniature soient capturés à partir de la vidéo pendant le transcodage, mais il s'agit de la valeur par défaut si le paramètre le profil sélectionné inclut les formats associés d'image. (S'il n'y a aucun format associé d'image dans le profil, la valeur par défaut pour capture-images est false.)

    Exemple de données de demande :

          {
        "master": { "url": "http://learning-services-media.brightcove.com/videos/mp4/Bird_Titmouse.mp4" },
        "profile": "multi-platform-standard-static",
        "capture-images": true
      }
    Notez que vous devez inclure le profile champ et spécifier un profil avec des formats associés d'image (tous les profils standard les possèdent).

    Images dans les profils d'ingest

    Voici des détails sur la façon dont les formats associés d'image sont spécifiés dans les profils d'ingest.

    Propriété Montant
    media_type "image"
    label « poster » ou « vignette » (vous avez besoin d'un rendu pour chacun)
    format « png » ou « jpg »
    width nombre (pixels)
    height nombre (pixels)

    Voici des exemples de données pour les deux formats associés :

                {
              "media_type": "image",
              "format": "jpg",
              "label": "poster",
              "width": 1280,
              "height": 720
            },
            {
              "media_type": "image",
              "format": "jpg",
              "label": "thumbnail",
              "width": 160,
              "height": 90
            }

    Vous pouvez spécifier png ou jpg comme format - les images actuellement capturées sont toujours sorties sous forme d'images jpg.

                {
                "height": 360,
                "label": "poster",
                "width": 480,
                "media_type": "image",
                "format": "png"
            },
            {
                "height": 90,
                "label": "thumbnail",
                "width": 160,
                "media_type": "image",
                "format": "png"
            }

    Récupération des données d'image

    Vous pouvez récupérer des données d'image à l'aide de l' API CMS.

    Le CMS API retournera les données d'image dans une GET requête https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id} ou, si vous voulez uniquement les données d'image, à https://cms.api.brightcove.com/v1/accounts/{account_id}/videos/{video_id}/images

    Dans les deux cas, les données d'image dans la réponse ressembleront à ceci :

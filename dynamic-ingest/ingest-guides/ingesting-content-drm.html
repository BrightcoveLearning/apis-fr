Aperçu

DRM (Digital Rights Management) protège votre contenu vidéo en cryptant les données vidéo et en les déverrouillant en fonction des politiques de licence. Lorsque le contenu vidéo est chargé dans un lecteur Brightcove, le lecteur rappelle un serveur de licences et obtient l'autorisation de lire la vidéo. Cette transaction est transparente pour les téléspectateurs.

Pour un aperçu général de la prise en charge par DRM et Video Cloud, consultez Aperçu: Gestion des droits numériques dans Video Cloud.

Remarques:
  • Pour utiliser le cryptage DRM ou HLS, votre compte doit être activé pour ces fonctionnalités. Contactez votre gestionnaire de compte pour plus d'informations.
  • Lorsque les vidéos sont activées pour les DRM, les sources/rendus MP4 sont automatiquement masqués (ce qui signifie qu'ils existent, mais ne sont pas envoyés au lecteur en tant que sources) dans Studio, l'CMS API et le Playback API.

  • Les DRM sont actuellement pris en charge dans les navigateurs OS mobiles suivants:

    • Safari iOS (11.2+)
    • Android Chrome

    Widevine est pris en charge pour les applications Android et FairPlay est pris en charge pour les applications iOS. Pour plus de détails sur la création d'applications mobiles, consultez le Kits de développement logiciel (SDK) Brightcove Native Player sommaire.

  • Brightcove ne prend actuellement pas en charge la lecture DRM uniquement vidéo ou audio.

DRM pour la livraison dynamique

Si votre compte est activé pour Livraison dynamique et il a été activé pour DRM par votre gestionnaire de compte, toutes les vidéos nouvellement téléchargées seront protégées. Si vous souhaitez appliquer une protection à des Livraison dynamique vidéos, veuillez contacter l'assistance Brightcove.

Si vous avez des vidéos plus anciennes dans un compte Dynamic Delivery qui ont été ingérées à l'aide du système d'ingestion hérité (pas de livraison dynamique), ils devront être retranscodés à l'aide d'un profil de livraison dynamique avant que DRM ne soit appliqué. Vidéos existantes déjà ingérées pour la diffusion dynamique n'ont pas besoin d'être retranscodés , mais comme indiqué ci-dessus, vous devez contacter le support et leur demander d'appliquer la protection DRM à ces vidéos.

Si votre compte est activé pour les DRM et que vous souhaitez diffuser certains titres sans DRM, définissez le drm_disabled drapeau via CMS API à true en utilisant un Mettre à jour la demande de vidéo.

Demande d'échantillon

Demander

La demande est PATCH demande à:

              https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id

Corps de la demande

              {
                  "drm_disabled": true
              }

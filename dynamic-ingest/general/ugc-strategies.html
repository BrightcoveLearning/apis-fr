Aperçu

Le problème clé dans le travail avec le contenu généré par l'utilisateur (UGC ; également connu sous le nom de Consumer-Generated Media, ou CGM) est de savoir comment obtenir la vidéo de vos utilisateurs dans votre compte Video Cloud. Évidemment, vous ne pouvez pas donner à tous ceux qui visitent votre site Web l'accès à votre compte Video Cloud Studio. Vous devez diriger les vidéos générées par les utilisateurs vers un serveur que vous contrôlez, puis ingérer ces vidéos sur votre compte Video Cloud de manière contrôlée à l'aide de Dynamic Ingest.

Principales étapes

  1. Planifiez votre campagne UGC. Formulez votre appel à l'action et vos politiques. Donnez des conseils à vos utilisateurs sur les problèmes de format de fichier et d'encodage vidéo. Tenez compte des problèmes de droits d'auteur et de marques de commerce.
  2. Mettre en place un serveur intermédiaire pour héberger votre formulaire de téléchargement et recevoir les vidéos.
  3. Créer un formulaire de téléchargement où vos utilisateurs peuvent télécharger leurs vidéos et métadonnées.
  4. Revoir les vidéos avant leur publication. Appliquez le filtre éditorial de votre choix aux vidéos générées par les utilisateurs.
  5. Ingérez les vidéos générées par l'utilisateur depuis le serveur intermédiaire vers votre bibliothèque multimédia Video Cloud. Vous pouvez utiliser Intégration dynamique pour transférer vos vidéos dans Video Cloud, ou utilisez API de téléchargement de fichier source pour Dynamic Ingest API si vous ne pouvez pas utiliser un référentiel à partir duquel Dynamic Ingest peut extraire directement.
  6. Publier votre choix des vidéos générées par les utilisateurs sur vos pages Web. Vous pouvez utiliser des listes de lecture intelligentes pour le contenu dynamique.

Planifiez votre campagne UGC

Vous devez planifier votre campagne UGC en fonction de vos objectifs commerciaux. Planifiez la façon dont vous communiquerez votre campagne UGC à votre public. Vous pouvez utiliser des promotions autour de votre contenu vidéo et de vos propriétés Web existants, ainsi que de ceux de vos affiliés et partenaires commerciaux. De nombreuses campagnes UGC ont un élément de concours, auquel cas vous devrez penser aux prix, au jugement et à la durée de la campagne.

Dans votre campagne UGC, vous devez communiquer vos politiques à vos utilisateurs. Quelques facteurs à considérer :

  • formats de fichiers acceptés
  • recommandations d'encodage du fichier source
  • taille maximale du fichier et longueur de la vidéo
  • les exigences légales, telles que la propriété et les droits de distribution de la vidéo téléchargée, et les restrictions imposées aux utilisateurs' l'utilisation des droits d'auteur et des marques (les vôtres et ceux de tiers).

Vous devrez également planifier la façon dont vous voulez publier les vidéos.

Examiner des vidéos avant de les publier

Dans une campagne UGC accessible au public, certains utilisateurs peuvent soumettre du contenu vidéo dont la publication est inacceptable - un contenu qui viole les droits d'auteur ou les marques de commerce, ou qui est obscène, offensant ou tout simplement inutile. Planifiez la manière dont vous souhaitez sélectionner ou modérer et publier les vidéos générées par les utilisateurs. Vous pouvez adopter plusieurs approches :

  • Passez en revue toutes les vidéos soumises sur votre serveur intermédiaire, puis déplacez celles qui sont acceptables dans un répertoire distinct à partir duquel elles peuvent être téléchargées vers votre bibliothèque multimédia Video Cloud.
  • Ingérez toutes les vidéos soumises directement dans votre bibliothèque multimédia Video Cloud, mais laissez-les dans un état inactif jusqu'à ce qu'elles aient été examinées et approuvées.

Mettre en place un référentiel intermédiaire

Vous ne voulez pas que le contenu généré par les utilisateurs aille directement dans votre bibliothèque multimédia Video Cloud, à moins que vous ne puissiez vraiment faire confiance à tous vos utilisateurs. Au lieu de cela, vous devez utiliser un référentiel intermédiaire: les utilisateurs téléchargent leurs vidéos dans le référentiel intermédiaire, puis vous utilisez Dynamic Ingest pour déplacer les vidéos du serveur intermédiaire vers votre bibliothèque multimédia Video Cloud.

Nous vous recommandons d'utiliser un compartiment S3 pour votre référentiel intermédiaire, car il est très facile de configurer Dynamic Ingest API à partir de S3, que le compartiment soit public ou protégé.

Créer un formulaire de téléchargement

Le formulaire de téléchargement permet aux utilisateurs de télécharger des vidéos sur votre serveur intermédiaire. Le formulaire de téléchargement comporte généralement des champs dans lesquels l'utilisateur peut fournir un nom, une description et des balises pour la vidéo, bien que vous puissiez ajouter un champ pour toute propriété de la vidéo que vous souhaitez modifier par l'utilisateur. (Vous pouvez également ajouter d'autres métadonnées lorsque vous ingérez la vidéo si vous le souhaitez.)

Ingérer la vidéo

Lorsque vous êtes prêt, vous pouvez ingérer le contenu UGC sur votre compte Video Cloud à l'aide de Dynamic Ingest. Deux requêtes API sont impliquées:

  1. UNE Créer une vidéo demande à l'CMS API, qui crée l'objet vidéo dans le système Video Cloud et ajoute des métadonnées telles que la description et les balises.
  2. Les Demande d'ingestion , qui extrait le fichier vidéo et le transcode.

Voir le Présentation de Dynamic Ingest API pour plus d'informations.

Publier le contenu

Réfléchissez à la manière dont vous souhaitez présenter vos vidéos générées par les utilisateurs. Sur quelles pages les publierez-vous ? Vous voulez un lecteur personnalisé ?

Vous pouvez regrouper automatiquement les vidéos générées par les utilisateurs à l'aide de listes de lecture intelligentes. Voici deux approches que vous pourriez envisager :

  • Faire une liste de lecture intelligente pour les vidéos soumises par les utilisateurs les plus vues. Pour ce faire, étiquetez toutes les vidéos soumises par les utilisateurs avec une étiquette commune et créez une nouvelle liste de lecture intelligente définie par "search": "tags:\" {YOUR_UGC_TAG}\" et faites le type de liste PLAYS_TOTAL
  • Définissez des balises basées sur le sujet et laissez vos utilisateurs sélectionner la meilleure.

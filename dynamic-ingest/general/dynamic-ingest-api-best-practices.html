introduction

Les API d'ingestion dynamique est puissant et flexible, mais pour l'utiliser plus efficacement, vous devez suivre quelques bonnes pratiques, qui sont détaillées ci-dessous.

Les meilleures pratiques

Vous trouverez ci-dessous les règles et directives relatives à l'ingestion dynamique.

  • Vous devriez ne pas avoir des tâches en double pour le traitement de la même vidéo, bien que l'ingestion d'autres actifs tels que des images et des pistes de texte avec la vidéo dans la même tâche soit acceptable.
    Si vous ingérez des images, des pistes de texte ou d'autres éléments pour la vidéo avant le traitement de la vidéo est terminé, ils ne pas être ajouté à la vidéo.

    si vous utilisez un processus automatisé pour créer des tâches d'ingestion qui pourraient potentiellement créer plusieurs tâches sur le même ID vidéo, vous devez vérifier avis pour s'assurer qu'il n'y a pas d'autres travaux en cours de traitement sur la même vidéo avant d'en créer un nouveau.

  • Les profile le champ pour les demandes d'ingestion est optionnel. Évitez de coder en dur le profile dans les applications et les intégrations - à la place, définissez le profil le plus couramment utilisé comme défaut pour le compte et omettre le champ, ou récupérer les profils disponibles pour le compte et forcer l'utilisateur à en choisir un.
  • Une limite de 20 requêtes par seconde sur chaque compte pour le nombre total de requêtes sur Intégration dynamique et le API CMS.
  • Il y a une limite de 100 tâches DI prioritaires simultanées par compte
  • Si vous ingérez de gros lots de fichiers, envisagez d'utiliser le File d'attente de faible priorité ce qui vous permettra de mettre en file d'attente plus de 100 travaux.
  • Le contenu ingéré est normalement limité à 100 Go par fichier (si vous devez ingérer des fichiers plus volumineux, veuillez contacter votre responsable de compte).
  • Si vous ingérez des fichiers par lots, limitez les tâches de priorité normale simultanées à 100 et attendez qu'une tâche termine le traitement avant d'en ajouter une autre - voir Stratégie de repli/réessayer pour plus d'informations. Alternativement, vous pouvez définir le "priorité": "faible" - en utilisant une priorité basse, vous n'avez pas à vous soucier de la taille de votre file d'attente d'ingestion, bien que le traitement puisse prendre un peu plus de temps.
  • Validez les modifications du profil d'ingestion pour vous assurer d'obtenir les résultats souhaités avant d'ingérer des lots complets.

Aperçu

Cet exemple et le code associé sont fournis comme guide pour le développement de votre production. Le code n'est pas certifié pour une utilisation en production ni officiellement pris en charge.
Notez que cette application a été conçue pour aider les clients de Brightcove à migrer de l'ancien Smart Player vers le nouveau Brightcove Player. Il a été construit autour de vidéos ajoutées à l'aide de l'ancien système d'ingestion et ne renvoie pas de résultats significatifs pour les vidéos Dynamic Delivery.

Cet outil génère un rapport qui comprend les éléments suivants :

  • Résumé montrant combien de vidéos ont non Rendus HLS ou MP4 ou les deux
  • Rapport pour chaque vidéo montrant les rendus HLS et MP4 (HTTP) (avec la plage de débit et la taille d'image pour chacun)

L'objectif est de vous aider à vous préparer à migrer du Smart Player vers le Brightcove Player en identifiant les vidéos qui doivent être transcodées pour créer des rendus HLS et/ou MP4.

Notez que les vidéos Dynamic Delivery ne sont pas cochées, car ce n'est pas nécessaire.

Obtenir des informations d'identification

Comme la plupart des API RESTful de Brightcove, le API CMS nécessite des jetons d'accès OAuth pour autoriser les demandes. Pour obtenir des jetons d'accès, vous devez d'abord obtenir les informations d'identification du client avec des autorisations pour les types d'opérations que vous souhaitez demander.

  1. Pour obtenir vos identifiants, suivez les instructions dans Gestion des informations d'identification de l'API. Il s'agit de la méthode recommandée pour obtenir vos informations d'identification client.     Vous pouvez également obtenir des informations d'identification à l'aide de l'API OAuth, en suivant les instructions de l'un des guides répertoriés ci-dessous :

  • OAuth : Obtenir les informations d'identification du client à l'aide de CURL
  • OAuth : Obtenez les informations d'identification du client à l'aide de Postman

Inspecteur de rendu

Vous pouvez utiliser l'application ci-dessous pour générer un rapport sur les rendus de toutes les vidéos de votre compte. Les vidéos sans rendu HLS ou HTTP MP4 (téléchargement progressif) devront être retranscodées pour fonctionner correctement dans le nouveau lecteur. Les vidéos avec uniquement des rendus HLS à faible débit doivent également être retranscodées.

Voir le stylo Inspecteur des rendus vidéo par Brightcove Learning Services (@rcrooks1969 ) au Stylo de code.

Code proxy

Afin de créer votre propre version de l'exemple d'application sur cette page, vous devez créer et héberger votre propre proxy. (Les proxys utilisés par Brightcove Learning Services n'acceptent que les demandes des domaines Brightcove.) Un exemple de proxy, très similaire à celui que nous utilisons, mais sans les contrôles qui bloquent les demandes des domaines non Brightcove, peut être trouvé dans ce référentiel GitHub. Vous y trouverez également des instructions de base pour l'utiliser et un guide plus élaboré pour créer des applications autour du proxy dans Utilisation des API REST.

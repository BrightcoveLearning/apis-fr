Champs principaux

Tableau 1: Champs principaux
Champ Taper Lecture seulement Défaut La description
id chaîne de caractères Oui rien l'identifiant de la playlist
account_id chaîne de caractères Oui rien l'identifiant du compte
created_at Chaîne date-heure ISO 8601 Oui rien la liste de lecture date-heure a été ajoutée au compte ; Exemple: "2014-12-09T06:07:11.877Z"
description chaîne de caractères non nul la brève description de la playlist - 248 caractères mono-octet maximum
favorite booléen non faux si la liste de lecture est incluse dans les favoris
limit numéro non 100 nombre de vidéos que la playlist peut contenir (entier, 1 - 100) renvoyé pour les listes de lecture intelligentes uniquement
name chaîne de caractères non rien Titre de la liste de lecture - champ obligatoire (longueur maximale : 248 caractères à un octet)
reference_id chaîne de caractères non nul toute valeur unique dans le compte
search chaîne de caractères non nul chaîne de recherche utilisée pour définir les vidéos d'une liste de lecture Smart (exemple : "+tags:"bird","birds"")
type chaîne de caractères Oui le type de liste de lecture Voir la note[1] au dessous de
updated_at Chaîne date-heure ISO 8601 Oui rien la liste de lecture date-heure a été modifiée pour la dernière fois ;[1-2] Exemple: "2015-01-13T17:45:21.977Z"
video_ids déployer non tableau d'identifiants pour les vidéos de la playlist Renvoyé pour les listes de lecture manuelles uniquement

Remarques

  • [1]

    Les type la valeur ne peut être que l'une des suivantes : EXPLICITE, ACTIVATED_OLDEST_TO_NEWEST , ACTIVATED_NEWEST_TO_OLDEST , ALPHABETICAL , PLAYS_TOTAL , PLAYS_TRAILING_WEEK , START_DATE_OLDEST_TO_NEWEST,START_DATE_NEWEST_TO_OLDEST

    EXPLICIT signifie qu'il s'agit d'une liste de lecture manuelle ; tous les autres types sont des listes de lecture intelligentes avec des noms correspondant à leur schéma de commande.

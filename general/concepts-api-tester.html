Bienvenue au testeur d'API Brightcove Learning Services. Le testeur peut être utilisé pour toutes les API de plate-forme qui utilisent OAuth pour demandes d'authentification

Obtenir des informations d'identification

Pour obtenir un client_id et client_secret , vous devrez accéder à l'interface utilisateur OAuth et enregistrer cette application:

  • Lecteur Brightcove: //studio.brightcove.com/products/player/admin/oauthsettings
  • Nuage vidéo: //studio.brightcove.com/products/videocloud/admin/oauthsettings

Vous pouvez également obtenir vos informations d'identification via CURL ou Postman - voir:

  • Obtenir les informations d'identification du client à l'aide de CURL
  • Obtenez les informations d'identification du client à l'aide de Postman

Utiliser le CodePen

Voici quelques conseils pour utiliser efficacement le CodePen ci-dessus:

  • Basculez l'affichage réel du lecteur en cliquant sur le bouton Résultat bouton.
  • Clique le HTML/CSS/JS pour afficher UN des types de code.
  • Cliquez sur Modifier sur CodePen dans le coin supérieur droit pour insérer ce CodePen dans votre propre compte.
  • Retrouvez tout le code associé à cet exemple dans ce Dépôt GitHub.

Code proxy

Afin de créer votre propre version de l'exemple d'application sur cette page, vous devez créer et héberger votre propre proxy. (Les proxys utilisés par Brightcove Learning Services n'acceptent que les demandes des domaines Brightcove.) Vous pouvez télécharger deux versions de notre code proxy:

  • Il s'agit d'une version générale qui s'attend à ce que les informations d'identification du client soient transmises avec la demande
  • Cette version vous permet d'enregistrer vos identifiants client dans le proxy lui-même sur les lignes 25-26 (recommandé)

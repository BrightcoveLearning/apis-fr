Lorsque vous utilisez les API Video Cloud, vos appels de méthode nécessiteront des jetons d'accès à moins que vous n'activiez l'authentification de base dans les paramètres du service du lecteur. Pour plus d'informations sur l'activation de l'authentification de base, consultez Gestion des paramètres de compte.

Noter: Cette fonctionnalité n'est disponible que pour les comptes Professional et Enterprise.

Création des inscriptions clients

La page d'authentification de l'API est utilisée pour créer de nouveaux enregistrements de clients. Lors de la création des inscriptions, vous pouvez sélectionner le service et les rôles qui seront activés avec l'inscription.

Pour créer une nouvelle inscription client pour votre compte:

  1. Clique le ADMINISTRER lien dans l'en-tête du Studio.
  2. Clique le Authentification API relier. La page d'authentification de l'API s'ouvrira et affichera vos enregistrements clients actuels.
    Noter: Seules les inscriptions des clients pour le compte auquel vous êtes connecté s'afficheront.
  3. Cliquez sur Enregistrer une nouvelle demande.
  4. Entrez un Nom et La description pour l'inscription du client.
  5. Sélectionnez un ou plusieurs de vos comptes pour l'autorisation.
  6. Sélectionnez les API et les autorisations à activer pour cet enregistrement.
    Noter: La sélection d'un grand nombre de comptes et d'API peut entraîner une taille importante et invalide de l'en-tête HTTP. La taille d'en-tête maximale acceptée par notre serveur est de 8 Ko, ce qui signifie que vous devez sélectionner 30 comptes ou moins auxquels appliquer les informations d'identification. Pour des raisons de sécurité, nous vous recommandons de limiter l'étendue des autorisations pour un ensemble d'informations d'identification au plus petit ensemble de comptes et d'opérations nécessaires.
  7. Cliquez sur sauvegarder.
  8. UNE identité du client et Secret du client sera retourné.
    Noter: Copiez le secret client et stockez-le dans un emplacement sécurisé en tant que secret client Ne fera pas être à nouveau affiché.

Gestion des inscriptions clients

La page Authentification API peut être utilisée pour afficher et gérer les inscriptions des clients. Clique le ADMINISTRER lien dans l'en-tête Studio, puis cliquez sur Authentification API. Une liste de vos inscriptions actuelles s'affichera.

Pour afficher les détails de l'inscription, cliquez sur le nom. À partir de là, vous pouvez:

  • Modifier le nom
  • Copiez l'identifiant client
  • Ajouter des comptes pour autorisation
  • Sélectionnez des API supplémentaires à activer
  • Supprimer l'identifiant

Cliquez sur sauvegarder pour enregistrer vos modifications.

Notez que si des informations d'identification sont créées pour plusieurs comptes, les utilisateurs ne verront ces informations d'identification que s'ils ont accès à tous ces comptes. Par exemple, l'utilisateur 1 crée des informations d'identification pour les comptes A et B. L'utilisateur 2 se connecte et n'a accès qu'au compte A. L'utilisateur 2 ne verra pas les informations d'identification car il n'a pas accès au compte B.

Supprimer les inscriptions des clients

Pour supprimer un enregistrement client, cliquez sur l'icône de suppression () à côté de l'enregistrement ou modifiez l'enregistrement et cliquez sur Supprimer l'enregistrement. La suppression d'un enregistrement le désactive définitivement et ne peut pas être annulé.

Travailler avec DFP

Si vous disposez actuellement de plusieurs comptes Video Cloud configurés en tant que sources de contenu dans DFP, vous avez deux options:

  1. Vous pouvez créer un ID client et un secret client uniques pour chaque compte Video Cloud configuré comme source de contenu dans DFP. Par exemple, si vous avez 10 comptes Video Cloud, vous aurez 10 applications nommées DFP dans vos paramètres d'authentification API dans Video Cloud.
  2. Vous pouvez créer un identifiant client et un secret client uniques pour tous les comptes Video Cloud qui ont été configurés en tant que sources de contenu dans DFP. Par exemple, si vous avez 10 comptes Video Cloud, vous auriez 1 application nommée DFP dans vos paramètres d'authentification API dans Video Cloud.

Que vous choisissiez l'option 1 ou 2, pour chaque source de contenu que vous configurez dans DFP, vous devez fournir un ID de compte unique. Ainsi, vous ne réduisez pas le nombre de sources de contenu actuellement configurées dans DFP ; vous remplacez simplement le jeton de lecture par l'ID client et le secret client, qui peuvent être identiques dans toutes les sources de contenu ou uniques pour chacune.

L'option n°1 prend plus de temps au départ. Cependant, si l'ID client et le secret client sont les mêmes dans toutes les sources de contenu (comme avec l'option #2), il sera difficile d'ajouter une nouvelle source de contenu car vous devrez générer un nouvel ID client et un nouveau secret client et mettre à jour toutes les sources de contenu dans DFP. Ainsi, Brightcove recommande l'option #1.

Clause de non-responsabilité

Cet exemple d'application vous permet d'explorer les bases de l'API Player Management. Il ne s'agit pas d'une application déployable prête pour l'entreprise. Par exemple, la validation des formulaires n'est pas exhaustive et les mots de passe sont envoyés NON via HTTPS.

Pour en savoir plus sur l'approche générale concernant l'architecture de cet exemple d'application, en particulier l'authentification et les implémentations AJAX, consultez le exemples qui utilisent HTTPRequest.

Comprendre les informations last_viewed

Chaque lecteur est associé à un tableau qui contient un enregistrement du nombre de fois qu'un lecteur a été chargé. Ces informations sont stockées dans un nom de tableau last_viewed. Vous voyez ci-dessous un exemple d'un tel tableau. Notez que le count enregistre combien de fois dans la journée le lecteur spécifique a été chargé. Les date spécifie le jour spécifique.

    [
      {
        "count": 13,
        "date": "2018-10-24T05:00:00.000Z"
      }, {
        "count": 22,
        "date": "2018-10-23T05:00:00.000Z"
      }, {
        "count": 40,
        "date": "2018-10-22T05:00:00.000Z"
      }, {
        "count": 2,
        "date": "2018-10-21T05:00:00.000Z"
      }
    ]
Notez ce qui suit:
  • Le décompte est effectué une fois par jour.
  • Le décompte est pour les charges de Lecteurs, il n'est pas nécessaire qu'une vidéo ait été réellement lue pour l'ajouter au décompte.
  • Le tableau est limité à 30 entrées. Il ne s'agit probablement PAS de 30 jours consécutifs, mais de 30 jours où le nombre n'était pas nul.
  • Les données compilées n'enregistrent PAS les dénombrements avant la mi-octobre 2018.
  • Si un Lecteur n'a pas de données, le tableau est vide.

Architecture de l'application

La logique d'application de haut niveau pour cette application est la suivante:

  1. Recueillir les données utilisateur nécessaires pour récupérer les informations de tous les Lecteurs d'un compte
  2. Construire la structure de données JSON à transmettre à l'API pour la création du lecteur
  3. En cliquant sur le bouton, effectuez un appel AJAX pour récupérer les informations de tous les Lecteurs d'un compte
  4. En cas de succès de l'appel AJAX, extrayez le dernière_vue tableau pour chaque Lecteur.
  5. Pour chaque Lecteur, affichez le dernière_vue informations dans le tableau

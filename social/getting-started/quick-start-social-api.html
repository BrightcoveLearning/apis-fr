introduction

L'API sociale vous permet d'obtenir le statut et l'historique du partage de vidéos sur les plateformes de médias sociaux. Ce démarrage rapide vous permet de voir les demandes disponibles et leurs réponses, en utilisant soit votre propre compte, soit un exemple de compte Brightcove Learning Services.

Remarques sur l'exemple d'application

  • Ceci est un démarrage rapide, destiné à vous donner un modèle pour créer une application autour de l'API sociale. Il vous permet de tester les requêtes de base, mais n'utilise aucun des paramètres que ces requêtes peuvent utiliser. Pour les voir, regardez le Référence API.
  • L'application fait également des requêtes au API CMS pour obtenir les noms des vidéos renvoyées par la demande initiale de statut de partage social vidéo. Cela sert uniquement à renseigner les sélecteurs vidéo pour les autres demandes, car l'API sociale ne renvoie que les identifiants vidéo, pas leurs noms.
Cet exemple et le code associé sont fournis comme guide pour le développement de votre production. Le code n'est pas certifié pour une utilisation en production ni officiellement pris en charge.

Obtenir des informations d'identification

Pour utiliser l'API sociale, vous aurez besoin des informations d'identification appropriées.

Le moyen le plus simple de les obtenir est d'utiliser le Studio Authentification administrateur/API rubrique, cliquez sur Enregistrer une nouvelle demande , et sélectionnez les autorisations sociales pour vos informations d'identification :

Voici quelques conseils pour utiliser efficacement le CodePen ci-dessus :

  • Basculez l'affichage réel du lecteur en cliquant sur le bouton Résultat bouton.
  • Clique le HTML/CSS/JS pour afficher UN des types de code.
  • Cliquez sur Modifier sur CodePen dans le coin supérieur droit pour insérer ce CodePen dans votre propre compte.
  • Retrouvez tout le code associé à cet exemple dans ce Dépôt GitHub.

Code proxy

Afin de créer votre propre version de l'exemple d'application sur cette page, vous devez créer et héberger votre propre proxy. (Les proxys utilisés par Brightcove Learning Services n'acceptent que les demandes des domaines Brightcove.) Un exemple de proxy, très similaire à celui que nous utilisons, mais sans les contrôles qui bloquent les demandes des domaines non Brightcove, peut être trouvé dans ce référentiel GitHub. Vous y trouverez également des instructions de base pour l'utiliser et un guide plus élaboré pour créer des applications autour du proxy dans Utilisation des API REST.

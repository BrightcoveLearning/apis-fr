L'API Brightcove Syndication Feed est une API publique permettant de récupérer les flux MRSS créés à l'aide du API de configuration de syndication.

L'API de configuration de syndication peut également être utilisée pour récupérer des flux en ouvrant simplement le syndication_url renvoyé dans l'objet de syndication dans un navigateur, mais l'API de flux est différente en ce sens qu'elle ne nécessite aucune authentification - toute personne disposant de la bonne URL peut récupérer le flux.

Les principaux cas d'utilisation pour récupérer des flux à l'aide d'une API ou d'une autre sont les suivants:

  • Si vous souhaitez uniquement héberger le flux vous-même, pour les abonnés, par exemple, vous utiliserez l'API de configuration de syndication pour récupérer le flux.
  • Si vous avez des partenaires de syndication que vous souhaitez donner au flux à utiliser et à héberger de la manière qu'ils souhaitent, vous pouvez donner l'URL de l'API de flux avec le syndication_id pour récupérer directement les flux.

Référence API / URL de base

Les Référence de l'API de flux contient tous les détails sur les demandes et les réponses disponibles.

L'URL de base est:

https://social.feeds.brightcove.com/v1

Authentification

L'accès à l'API Feed ne ne pas nécessitent une authentification, car les flux sont destinés à être accessibles au public.

Opérations

Il n'y a qu'une seule opération pour l'API Feed: OBTENEZ un flux basé sur le syndication_id.

URL de demande

    https://social.feeds.brightcove.com/v1/accounts/{account_id}/mrss/syndications/{syndication_id}/feed

Méthode: AVOIR

Paramètres de requête

Tous les paramètres de requête sont facultatifs.

limit
Nombre maximal d'éléments à inclure dans la réponse au flux. Plage autorisée de 1 à 100. Défaut: 100.
offset
Nombre d'éléments à ignorer lors de la génération du flux renvoyé. Défaut: 0.
updated_after
Une chaîne datetime ISO-8601 (par exemple, 2016-03-16T15:42:06.000Z ou 2016-03-16T10:47:29-05:00 ) qui limitera les résultats du flux aux éléments mis à jour après la date indiquée.

Corps de la requête: aucun

C'est important! si vous incluez un corps de requête, une erreur sera renvoyée.

Type de réponse: XML

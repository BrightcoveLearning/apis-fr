Aperçu

REMARQUE: La méthode préférée pour obtenir les informations d'identification du client consiste à utiliser l'interface utilisateur d'administration de Studio. Ce document est destiné à ceux qui choisissent d'utiliser OAuth API à la place.

La meilleure et première étape pour comprendre les informations contenues dans ce document est d'examiner les Présentation d'OAuth si vous ne l'avez pas encore fait.

Ton BC_TOKEN est un identifiant unique et anonyme pour vous en tant qu'utilisateur Brightcove. En dehors de son existence dans un système Brightcove backend, il est défini comme un cookie lorsque vous vous connectez à Studio afin que Studio connaisse votre rôle et les comptes auxquels vous avez accès. Tout cela est transparent et ne nécessite aucune action de votre part.

L'utilisation du BC_TOKEN pour vous consiste à authentifier une demande d'API OAuth pour créer des informations d'identification client.

Remarques:
  • Les BC_TOKEN value n'a aucun rapport avec les jetons de gestion des API que vous traitez dans Studio.
  • Si tu utiliser l'administrateur Studio pour créer les informations d'identification du client , vous n'avez pas besoin de récupérer votre BC_TOKEN - Studio s'en chargera pour vous.
  • Les BC_TOKEN cookie est un cookie de session et expire à chaque fois que vous vous déconnectez de Studio (il expire également automatiquement après 24 heures). A chaque fois un nouveau BC_TOKEN cookie est supprimé lorsque vous vous connectez à Studio, il aura une nouvelle valeur.

Conditions

  • Un compte Brightcove valide avec des autorisations d'administrateur requises pour créer des informations d'identification client

introduction

le Playback API comporte quatre opérations:

  • Obtenir une vidéo par identifiant
  • Obtenir une vidéo par identifiant de référence
  • Obtenir une playlist par identifiant
  • Obtenir une liste de lecture par identifiant de référence
le Playback API est destinée à une utilisation côté client par des pages Web ou des applications mobiles - pour la gestion générale des médias ou l'intégration de Video Cloud avec vos systèmes, utilisez le CMS API au lieu.

Si vous souhaitez uniquement des informations sur la vidéo actuellement dans le lecteur, vous pouvez utiliser le mediainfo objet au lieu de faire un appel API Playback.

Les mediainfo L'objet contient des informations sur la vidéo actuellement chargée dans le lecteur, tandis que le Playback API peut récupérer des informations sur n'importe quelle vidéo. Pour plus de détails, consultez le Métadonnées vidéo de mediainfo document.

Authentification

Les demandes d'Playback API sont authentifiées à l'aide d'une chaîne de clé de stratégie chiffrée. Le CodePen ci-dessous contient une clé de stratégie pour un exemple de compte Brightcove, vous pouvez donc l'utiliser si vous le souhaitez. Vous pouvez obtenir des clés de stratégie pour votre compte en utilisant soit le Policy API ou de l'un de vos Lecteurs Brightcove.

Instructions

Utilisez le CodePen ci-dessous pour explorer l'API:

  1. Commencez par faire simplement des demandes de vidéo et de liste de lecture pour un exemple de compte Brightcove pour voir les résultats.
  2. Ensuite, essayez d'obtenir votre propre clé de stratégie en utilisant soit le Policy API ou de l'un de vos Lecteurs Brightcove.
  3. Dans le formulaire CodePen ci-dessous, remplacez l'identifiant du compte, la clé de stratégie, l'identifiant vidéo et l'identifiant de la playlist par vos propres valeurs.
  4. Essayez de modifier le CSS pour personnaliser l'affichage des données vidéo.
  5. Essayez de modifier le JavaScript pour récupérer les vidéos et la liste de lecture par identifiant de référence au lieu d'identifiant.

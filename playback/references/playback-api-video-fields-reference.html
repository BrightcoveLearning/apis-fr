key_systems Objet

L'objet key_systems contient une liste d'objets, un pour chaque nom de clé DRM. Si cet objet est défini, alors sa source est protégée par le contenu.

Tableau 4 : objet key_systems
Champ Taper Défaut La description
nom de la clé Objet nul Nom de l'objet qui spécifie le type de DRM – les clés valides incluent:

* Fair-play: com.apple.fps.1_0
* Prêt à jouer: com.microsoft.playready
* Widevine: com.widevine.alpha
FairPlay utilise key_request_url et certificate_url
d'autres utilisent license_url		 Chaîne de caractères nul URL de l'emplacement de votre licence ou certificat DRM
 

Exemple 1:

    "key_systems": {
      "com.apple.fps.1_0": {
        "key_request_url": "https://....",
        "certificate_url": "https://..."
      }
    }

Exemple 2 :

    "key_systems": {
      "com.microsoft.playready": {
        "license_url": "https://..."
      },
      "com.widevine.alpha": {
        "license_url": "https://..."
      }
    },

variant objet

chaque variant contient une version des métadonnées vidéo pour une langue particulière.

Tableau 5 : variante Objet
Champ Taper Défaut La description
language Chaîne de caractères rien La langue de la variante sous la forme langue-PAYS, telle que ja-JA
name Chaîne de caractères rien Le titre traduit
description chaîne de caractères rien La description traduite
long_description chaîne de caractères rien La longue_description traduite
custom_fields objet {} Une carte des paires clé-valeur pour les champs personnalisés

Exemple

{
  "language": "ja-JA",
  "name": "バーゼル：ライン川",
  "description": "スイス、バーゼルの中心部にあるライン川のパノラマビュー。",
  "long_description": "スイス、バーゼルの中心部にあるライン川のパノラマビュー。",
  "custom_fields": {
    "language": "Japanese"
  }
}

introduction

Le schéma de métadonnées publicitaires (ad_server_data) prend en charge une syntaxe JSON plate sans éléments imbriqués (paires clé-valeur uniquement). L'ensemble de données du serveur publicitaire via l'API est utilisé pour toutes les coupures publicitaires reçues après la mise à jour jusqu'à ce que les données soient écrasées ou réinitialisées. Les clients peuvent mettre à jour et/ou réinitialiser les données à tout moment pendant leur diffusion en direct.

Ces métadonnées ne persistent pas entre les activations Steam. Les métadonnées publicitaires qui doivent persister entre les activations de flux doivent être définies dans la configuration publicitaire de l'application SSAI. L'ensemble de données via l'API est configurable pour une utilisation en tant que macros sur le serveur publicitaire et le remplacement des balises côté serveur.

Les métadonnées du programme publicitaire sont transmises au système SSAI via des balises de liste de lecture personnalisées et suivent la priorité des paramètres publicitaires dynamiques (DAP) existants dans le système SSAI. Les données du serveur publicitaire envoyées sur un point de repère ont priorité sur les métadonnées attribuées à la tâche. Si des collisions de clés se produisent entre les données de point de repère et les métadonnées de l'annonce sur le travail, les valeurs de point de repère sont considérées comme prioritaires et utilisées dans la collection fusionnée

Les métadonnées publicitaires fournies sur un point de repère ne sont utilisées que pour ce point de repère et ne persistent pas pour les insertions de point de repère suivantes. Seules les métadonnées publicitaires définies via l'API de métadonnées publicitaires du programme persisteront pour toutes les coupures publicitaires reçues par la suite jusqu'à ce qu'elles soient mises à jour ou réinitialisées.

Points de terminaison

Pour plus de détails sur les points de terminaison, consultez le Référence de l'Live API.

Ajouter/mettre à jour

Pour ajouter ou mettre à jour les métadonnées des annonces de programmes en direct, envoyez un POST demande à:

https://api.bcovlive.io/v1/jobs/{job_id}/cuepointdata

Ajouter/mettre à jour des champs
Champ Taper La description Exemple
ad_server_data Objet Un objet contenant des paires clé/valeur. 
{
  "ad_server_data": {
  	"one":"won",
  	"two":2,
  	"three":false
  }
}

Effacer

Pour supprimer les métadonnées d'annonces de programmes en direct, envoyez un DELETE demande à:

https://api.bcovlive.io/v1/jobs/{job_id}/cuepointdata

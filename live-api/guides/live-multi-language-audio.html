Conditions

  • Un compte configuré pour l'API Live avec la fonction audio multilingue activée (contactez votre gestionnaire de compte pour plus de détails)
  • Une clé API pour l'API Live (fournie lors de la création du compte)
  • Un encodeur qui prend en charge le protocole RTP. La prise en charge audio multilingue est limitée à rtp ou rtp-fec protocole.
  • Le PID de chaque piste audio doit être connu lors de la création du Job.

Créez votre emploi en direct

Afin d'ajouter plusieurs pistes audio à votre tâche, une liste de pistes audio doit être fournie dans le corps de la demande lors de la création d'une tâche en direct :

{
  "live_stream": true,
  "region": "us-west-2",
  "protocol": "rtp",
  "cidr_whitelist": [ /* omitted… */ ],
  "outputs": [ /* omitted… */ ],
  "alternate_audio": {
    "tracks": [ { Track Details } ]
  }
}

Champs pris en charge pour le Track objet

Le tableau ci-dessous contient une description complète des champs pris en charge dans le track objet

Suivre les champs
Champ La description
pid Obligatoire. Le PID à choisir dans le TS entrant pour la piste qui est spécifié dans le fichier de flux envoyé à Wowza.
language Obligatoire. Le code de la langue à utiliser. À l'heure actuelle, cela est flexible et pourrait être dans l'un des différents formats de langue requis/pris en charge par les spécifications, en particulier RFC5456/BCP47 qui couvre également ISO-639 selon les exigences. https://tools.ietf.org/html/rfc5646

HLS > LANGUAGE champ

TABLEAU DE BORD > AdaptationSet:lang champ
variant Optionnel. Correspond aux valeurs du schéma de rôle DASH : principal, alternatif, commentaire, supplémentaire, dub. Utilisé dans le rôle DASH urn:mpeg:dash:role:2011. Par défaut à main.

HLS > CARACTÉRISTIQUES voir https://tools.ietf.org/html/rfc8216#section-4.3.4.1
label Optionnel. Une description textuelle à utiliser pour la piste, de préférence à utiliser par le joueur dans la mesure du possible. Utilisé dans: HLS > NAME champ. Par défaut à la language avec un identifiant pour chaque piste par exemple en-0 si non spécifié.
streams[] Optionnel. Si la copie n'est pas présente, utilisez uniquement les paramètres audio par défaut.

HLS > GROUP-ID. (Objet)

Exemple: { “source”: “720p” } - correspond à l'étiquette de la source de sortie.

Cela imite le streams champ du type de sortie de la liste de lecture.
default Obligatoire sauf en cas de défaut. Quelle piste audio doit être marquée comme DEFAULT dans le EXT-X-MEDIA ainsi que ceux à multiplexer dans le flux vidéo *

S'il n'est pas présent, le premier est par défaut.

* si plusieurs listes de lecture sont définies avec des valeurs par défaut différentes, la piste par défaut de niveau supérieur est la piste multiplexée

Configurez votre encodeur

Configurez votre encodeur pour fournir MPEG-TS sur RTP à stream_url à partir de la réponse de création. Assurez-vous que les pistes audio fournies dans le corps de la demande sont configurées avec le bon PID dans votre encodeur.

Tester la lecture

Chargez le playback_url dans le Lecteur Brightcove ou Safari pour confirmer la lecture.

DRM_Lecture

Actuellement, DRM n'est pris en charge que via le service de rattrapage (timeshift). Ajoutez ces champs à votre corps de demande de demande de création d'emploi :

{
  "drm": {
    "modes": [
      "all"
    ]
  },
  "timeshift": {
    "type": "single",
    "startover_duration": 36000,
    "skip_live_output": true
  }
}

La réponse comprendra des sorties DRM et un timeshift_url à utiliser pour la lecture.

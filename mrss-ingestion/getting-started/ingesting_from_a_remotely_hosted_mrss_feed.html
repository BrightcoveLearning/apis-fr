introduction

Vous pouvez utiliser Media RSS (MRSS) pour que Brightcove extrait votre contenu de votre propre système à partir d'un flux MRSS que vous publiez. Video Cloud extrait les métadonnées du flux MRSS et crée soit un actif distant, soit des téléchargements (à partir de l'endroit où ils sont hébergés) et traite les vidéos et tous les actifs associés (miniatures, fichiers de sous-titres codés, etc.).

Si vous souhaitez participer à la version bêta de l'ingestion à partir d'un flux MRSS hébergé à distance, contactez votre responsable de compte.

Présentation de MRSS Ingest

À propos de MRSS

Media RSS (MRSS) est une extension de RSS qui vous permet de syndiquer des fichiers multimédias dans des flux RSS. Pour les détails techniques, voir le Spécification RSS des médias.

Configuration de MRSS

Contactez votre gestionnaire de compte et indiquez que vous souhaitez utiliser MRSS. Votre flux MRSS devra être hébergé sur une URL publique, et les vidéos et ressources qu'il contient doivent également être sur des URL publiques. Les URL peuvent inclure http:// ou https://.

Utilisation de MRSS

Pour utiliser MRSS, vous hébergez des métadonnées et/ou des vidéos et d'autres fichiers (vignettes, sous-titres codés) sur un serveur Web, et créez un fichier manifeste XML qui fait référence à ce contenu.

Notez que le flux MRSS sera extrait par Brightcove toutes les cinq minutes.

Utilisation de MRSS pour extraire uniquement les métadonnées

  1. Créez un fichier manifeste XML comme décrit dans fichier manifeste XML. Vous devez utiliser le ooyala:remoteasset balise pour l'ingestion de métadonnées uniquement.
  2. Hébergez le fichier manifeste XML sur l'un des emplacements MRSS que vous avez fournis. Assurez-vous que le nom de fichier et le chemin correspondent exactement à l'un des chemins que vous avez fournis à Brightcove.

Utilisation de MRSS pour extraire des vidéos, des vignettes et des fichiers de sous-titres codés

  1. Hébergez un ou plusieurs fichiers (vidéos, vignettes et fichiers de sous-titres codés) sur un serveur Web. Les URL doivent être accessibles au public.
  2. Créé un fichier manifeste XML. Assurez-vous que les entrées du fichier référencent les fichiers sur votre serveur. En particulier, la valeur de la <media:content> éléments href attribut doit être une URL HTTP vers le fichier sur votre serveur, comme dans l'exemple suivant: 
    <media:content url="http://mysite.com/upload/lacrosse_70.mov" />
  3. Hébergez le fichier manifeste XML sur l'un des emplacements MRSS que vous avez fournis. Assurez-vous que le nom de fichier et le chemin correspondent exactement à l'un des chemins que vous avez fournis à Brightcove.
    Noter: Malheureusement, le concept de « journalisation client » ne s'applique pas au traitement basé sur les flux MRSS. Le meilleur moyen de vérifier que votre flux est traité avec succès est de vérifier l'apparence de vos éléments, soit avec Studio Nuage Vidéo ou le Nuage Vidéo CMS API.

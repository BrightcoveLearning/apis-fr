Messages d'erreur

Code HTTP Code d'erreur La description Résolution
400 BAD_REQUEST Cette erreur indique que la demande effectuée comportait une erreur côté client. Le message contient généralement des informations sur la nature de l'erreur. Les message les champs de la réponse contiennent des informations sur la cause de l'erreur, telles que « valeur non valide pour sort paramètre.
400 UNSUPPORTED_FIELD_COMBINATION_ERROR Cette erreur indique que le fields paramètre d'un rapport comprenait un ou plusieurs champs qui ne sont pas valides pour le type de rapport. Les message Les champs de la réponse contiennent des informations sur les champs non valides qui ont été spécifiés.
400 TIME_RANGE_UNAVAILABLE Cette réponse est renvoyée lorsque le système de traitement historique/par lots n'a pas les données que vous demandez.
401 UNAUTHORIZED Cette erreur indique que l'autorisation (OAuth2 ou LA token) utilisée pour faire la demande n'est pas valide. Vérifiez le jeton d'autorisation utilisé. Il est possible que le token utilisé soit au-delà de la fenêtre de temps pour laquelle il était valide et qu'un token plus récent soit nécessaire. Généralement, chaque jeton expire après 5 minutes. Le jeton d'autorisation peut être spécifié en tant que paramètre d'URL access_token ou dans le Authorization entête. Faire référence à Gestion des informations d'identification de l'API pour plus de détails.
404 NOT_FOUND Cette erreur est généralement renvoyée lorsque le point de terminaison demandé n'existe pas. Vérifiez le point de terminaison demandé.
405 METHOD_NOT_ALLOWED Cette erreur se produit lorsque la requête api est effectuée avec une méthode HTTP autre que GET. Faire toutes les requêtes à l'API avec HTTP GET.
500 SERVER_ERROR Cette erreur se produit lorsqu'il y a une erreur interne sur le serveur. Le serveur peut le renvoyer lorsqu'il n'est pas disponible ou qu'il est confronté à un problème transitoire. Essayez à nouveau dans un certain temps avec un certain recul exponentiel. Si le problème persiste, veuillez créer un ticket d'assistance.
503 PROCESSING Cela signifie que le serveur a reçu la demande et la traite actuellement. Les Analytics API peut renvoyer ce message s'il rencontre une requête de longue durée. Une fois la requête terminée, elle sera stockée dans le cache du serveur pendant 5 minutes maximum. Par conséquent, nous vous suggérons d'interroger l'API 4 minutes après avoir reçu cette erreur

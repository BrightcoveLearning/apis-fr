    FAQ : API Analytics (Statistiques)

    Vous trouverez ci-dessous quelques questions fréquemment posées sur l'API Analytics. Pour obtenir des réponses à certaines des questions courantes concernant le module Analytics Video Cloud Studio, consultez la FAQ : Module Analytics.

    Plages de dates

    Comment puis-je obtenir des données pour une journée spécifique ?
    Réglez à la fois le from et to valeurs à cette date. Par exemple : from=2013-05-12&to=2013-05-12. En outre, vous pouvez interroger par la date dimension , qui peut être combinée avec d'autres dimensions.
    Le module Analytics et le module me Analytics API donnent-ils des données identiales pour la même plage de dates ?
    Oui, mais ils peuvent être différents si vous comparez des données relativement récentes et que vous obtenez des données de Studio et de l'API à des moments différents :
    1. Les données datent de moins de 3 jours. Les données analytiques datant de moins de 3 jours sont provisoires et peuvent être modifiées à tout moment.
    2. Les données pour la même période sont examinées lorsqu'elles étaient âgées de moins de 32 jours, puis après 32 jours. Après 32 jours, les données sont déplacées vers un référentiel « historique » distinct, et à ce stade, nous rejetons certaines données, en particulier les données d'engagement détaillées. Cela signifie que les champs calculés tels que le score d'engagement peuvent changer légèrement, car le calcul est effectué sur des données moins granulaires.

    Granularité des données

    Quelle est la granularité des données Analytics récentes ?
    Pour les 32 jours les plus récents (y compris le jour en cours), le Analytics API signale une granularité horaire, car il enregistre des valeurs dans des compartiments horaires. Cependant, l'heure actuelle est toujours remplie, ce qui peut donner à l'API l'apparence d'avoir plus de granularité qu'une heure lorsque vous demandez des données qui tombent dans l'heure actuelle.

    Par exemple :

    1. Si vous demandez des données d'il y a 3 heures pour 9:15 et 9:20, vous pouvez obtenir la même valeur si elles se trouvent dans le même compartiment horaire ; cependant, elles peuvent être différentes car toutes les données analytiques sont provisoires et sujettes à modification jusqu'à ce qu'elles soient rapprochées.
    2. Si vous demandez des données d'il y a 10 minutes, puis attendez 5 minutes et demandez à nouveau, vous pouvez obtenir une valeur différente même si elle se trouve dans le même compartiment, car ce compartiment est toujours en cours de mise à jour.
    Quel est l'intervalle d'actualisation des données analytiques ?
    Le point de terminaison /data est actuellement mis en cache pendant 5 minutes entre les requêtes, donc pour les listes de vidéos avec du trafic au cours de la dernière heure, les intervalles de 5 minutes sont le plus petit délai qui vous donnera un delta à partir de laquelle travailler.
    Quelle est la granularité des données analytiques historiques ?
    Pour les dates antérieures aux 32 derniers jours, les valeurs journalières complètes sont Analytics API signalées. Cela signifie que si votre demande from=1368334306919&to=1378446336919 (du dim. 12 mai 2013 04:51:46 GMT au Fri, 06 Sep 2013 05:45:36 GMT), vous obtiendrez les mêmes résultats que si vous aviez demandé from=2013-05-12&to=2013-09-06

    Eléments et champs

    Quelle est la meilleure façon de dire combien de débuts de flux il y avait au cours d'une période pour une ou plusieurs vidéos ?
    Il suffit de regarder la video_view métrique - ce sera toujours égal au flux commence. Vous pouvez voir cette mesure dans tous les rapports, et le résumé affichera les démarrages en streaming pour toutes les vidéos de la plage de dates interrogées. Pour voir les démarrages du flux pour toutes les vidéos de votre compte, vous devez simplement exécuter cette requête : 
        https://analytics.api.brightcove.com/v1/data?accounts={account_ids}&dimensions=video&limit=all&offset=0&fields=all
    Comment retourner tous les articles d'un rapport?
    Définissez le limit paramètre du rapport égal à all.
    Quelles fields sont les données renvoyées pour les éléments par défaut ?
    Par défaut, seul video_view et le champ correspondant à la dimension demandée (par exemple destination_domain ) sont renvoyés. Pour obtenir des champs supplémentaires, définissez fields=field_name1,field_name2 pour renvoyer les champs sélectionnés.
    Qu'est-ce que cela signifie si une valeur de champ est null?
    Une null valeur pour un champ de données indique que les données demandées n'ont pas été traitées. Les raisons les plus probables sont les suivantes :
    • Les données que vous avez demandées sont très récentes et n'ont pas encore été traitées
    • Les données demandées sont très anciennes et n'ont pas encore été importées dans le système d'analyse actuel
    J'ai changé le nom d'une de mes vidéos. Pourquoi est-ce que je vois l'ancien nom dans les données Analytics ?
    Le nouveau nom sera enregistré pour tous les nouveaux événements analytiques, mais nous ne modifions pas le nom dans les données historiques - le nom de la vidéo retourné sera le nom qu'il avait au moment où il a été visualisé.
    Pourquoi ne puis-je pas toujours voir les entiers pour les valeurs d'engagement ?
    Il est possible que les numéros d'engagement retournent avec des points décimaux. La raison en est que l'engagement est normalisé, ce qui signifie qu'il s'agit d'un rapport de « video_percent_view* (video_engagement_25/video_engagement_sum) », donc en fonction de la plage de temps sélectionnée pour la requête, vous pouvez voir des nombres à virgule flottante dans les cas où ils ne se divisent pas exactement.
    Comment filtrer les résultats par joueur ?

    Notez ce qui suit lorsque vous utilisez des where filtres sur les joueurs :

    • Brightcove Players - Entrez l'URL du lecteur dans ce format : players.brightcove.com/account id/playerID_default
    • Joueurs intelligents - Saisissez l'ID du joueur

    Rapports

    Pourquoi le compteur de bande passante de mon lecteur reste-t-il à 0 Go ?
    Dans le passé, Brightcove a signalé la bande passante par joueur, et donc pour les clients qui voulaient décomposer les choses par joueur, ils pouvaient utiliser ces mesures comme moyen d'allouer les coûts de bande passante. Toutefois, au fur et à mesure de la progression du marché vers une diffusion avec manifeste (HLS et bientôt DASH), la nature même de la vidéo segmentée rendra imprécis le suivi de bande passante par lecteur. Par conséquent, nous rendrons obsolète l'échelle de bande passante par lecteur dans le rapport Utilization. Nous vous recommandons donc d'utiliser les secondes vues par joueur dans le rapport Performance pour répartir les coûts par joueur.

    Divers

    Est-il possible d'exclure les vidéos visionnées par les employés des rapports de statistiques ?
    Cette tâche s'avère plutôt complexe et parmi les vidéos qui génèrent le plus de trafic, celles qui ont été visionnées par les employés représentent probablement un si petit pourcentage qu'elles ne risquent pas d'affecter les statistiques de façon significative. Toutefois, si vous pensez que cette exclusion est pertinente pour les vidéos qui génèrent peu de trafic, la solution la plus simple est de dupliquer vos ou votre lecteur de production et de demander aux employés de tester ou visionner les vidéos sur les copies de lecteur. Vous pourrez alors vous servir des filtres pour créer des rapports sur vos lecteurs de production uniquement, à l'aide de la fonctionnalité Rapports personnalisés du Module de statistiques ou de l'API de statistiques.

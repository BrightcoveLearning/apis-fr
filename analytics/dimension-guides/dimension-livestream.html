Points de terminaison d'analyse en direct

En plus de live_stream dimension couverte dans cette rubrique, il existe également des points de terminaison spéciaux pour récupérer des données d'analyse supplémentaires pour les tâches de streaming Brightcove Live. Voir le Référence de l'Analytics API pour les détails.

Paramètres d'URL

Les rapports de Analytics API prennent en charge les paramètres d'URL suivants.

Paramètres d'URL
Paramètre La description Obligatoire Valeurs Défaut
account Les comptes sur lesquels vous souhaitez créer un rapport Oui un ou plusieurs identifiants de compte sous forme de liste délimitée par des virgules rien
dimensions La ou les dimensions à signaler Oui une ou plusieurs dimensions sous forme de liste délimitée par des virgules (certaines combinaisons ne sont pas valides) rien
where Utilisé pour spécifier des filtres pour les rapports non {dimension}=={value} - un ou plusieurs sous forme de liste délimitée par des points-virgules rien

offset Nombre d'éléments à ignorer non entier positif 0

format Format pour renvoyer les résultats dans non json | csv | xlxs json
reconciled S'il est inclus, limitera les résultats aux données historiques ou en temps réel non vrai | faux vrai
from Le début de la plage de dates de la demande non Une date ISO 8601 (AAAA-MM-JJ), l'heure d'époque en millisecondes, la chaîne de tous les temps ou la date relative (-1m) ; seules les dates au cours des 32 derniers jours sont autorisées pour les points de terminaison d'engagement ou si réconcilié=false. 30 jours avant maintenant

Champs disponibles

Les champs suivants peuvent être renvoyés pour la dimension.

  • bytes_delivered- nombre total d'octets délivrés pour les flux en direct, y compris le lecteur et tous les médias associés
  • origin_bytes_delivered- octets délivrés pour les flux en direct eux-mêmes
  • live_stream- id du flux en direct, si disponible

    Dimension : device_os

    Dans cette rubrique, vous découvrirez les Analytics API device_os dimension. La device_type dimension fournit des informations sur le système d'exploitation du périphérique sur lequel les vidéos ont été visualisées.

    Paramètres d'URL

    Les rapports d'API Analytics prennent en charge les paramètres d'URL suivants.

    Paramètres d'URL
    Paramètre Description Obligatoire Valeurs Par défaut
    account Les comptes sur lesquels vous souhaitez signaler oui un ou plusieurs identifiants de compte en tant que liste délimitée par des virgules aucun
    dimensions La (les) dimension (s) à signaler oui une ou plusieurs dimensions en tant que liste délimitée par des virgules (certaines combinaisons ne sont pas valides) aucun
    where Utilisé pour spécifier des filtres pour les rapports non {dimension} == {value} - une ou plusieurs en tant que liste délimitée par des points-virgules aucun
    limit Nombre d'articles à retourner non entier positif 10
    offset Nombre d'éléments à ignorer non entier positif 0
    sort Champ pour trier les éléments sur non un champ valide pour la demande video_view
    fields Champs à renvoyer non varie en fonction de la dimension sur laquelle vous signalez video_view
    format Formater pour renvoyer les résultats dans non json | csv | xlxs Jason
    reconciled S'ils sont inclus, limiteront les résultats aux données historiques ou en temps réel non vrai | faux true
    from Début de la plage de dates pour la demande non Une date ISO 8601 (AAAA-MM-JJ), une heure d'époque en millisecondes, la chaîne alltime ou une date relative (-1m) ; seules les dates des 32 derniers jours sont autorisées pour les points de terminaison d'engagement ou si elles sont rapprochées = false. 30 jours avant maintenant
    to Fin de la plage de dates de la demande non Une date ISO 8601 (AAAA-MM-JJ), une heure d'époque en millisecondes, la chaîne maintenant ou des données relatives (+7d) ; seules les dates au cours des 32 derniers jours sont autorisées pour les points de terminaison d'engagement ou si rapproché = false. maintenant

    Champs disponibles

    Les champs suivants peuvent être renvoyés pour la dimension.

    • ad_mode_begin - le nombre de fois où le joueur est entré en mode publicitaire
    • ad_mode_complete - le nombre de fois où le joueur a terminé le mode publicitaire
    • device_os le système d'exploitation de l'appareil
    • engagement_score - le score d'engagement moyen d'une vidéo jouée dans un joueur
    • play_request - nombre total de fois que la lecture vidéo a été demandée
    • play_rate - le taux de lecture moyen (vues vidéo divisées par impressions) pour une vidéo
    • player_load - le nombre de fois où un joueur a été chargé
    • video_impression - le nombre total de vidéos chargées dans le lecteur
    • video_percent_viewed - la somme du pourcentage visionné des vidéos dans le lecteur
    • video_seconds_viewed - nombre total de secondes de vidéos vues dans le lecteur
    • video_view - nombre total de fois que la lecture vidéo a démarré dans le lecteur

    Valeurs de filtre

    Utilisée comme filtre, une cote permet de réduire les résultats.

    Valeurs de filtre : android, bada, ios, rim, symbian, web_os, windows, os_x, mac, linux, autre

    Combinaison des

    Pour voir quelles dimensions cette option peut être combinée et quels champs sont disponibles pour cette combinaison, reportez-vous à la section Dimension et champs.

    Exemples de demande

    Demande de dimension

        https ://analytics.api.brightcove.com/v1/data ? accounts=1752604059001&dimensions=device_os

    Demande de filtre

        https ://analytics.api.brightcove.com/v1/data ? accounts=1752604059001&dimensions=video&where=device_os==os_x, mac, ios

    Réponse

        La réponse apparaîtra ici lorsque vous testez une demande

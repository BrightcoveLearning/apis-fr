Limites

Cet exemple et le code associé sont fournis comme guide pour le développement de votre production. Le code n'est pas certifié pour une utilisation en production ni officiellement pris en charge.
  • Vous devez récupérer les données de la playlist et sélectionner une playlist avant de soumettre la demande à Analytics API
  • Pour les playlists intelligentes, les données d'analyse seront renvoyées pour les vidéos actuellement dans la playlist - les données ne seront pas renvoyées pour les vidéos qui ne sont plus dans la playlist car elles ont été désactivées, supprimées ou supprimées car la définition de la playlist a changé
  • Si l'une des vidéos est également publiée dans un seul lecteur vidéo ou dans plusieurs listes de lecture, vous devrez spécifier le lecteur pour obtenir des données précises pour une liste de lecture spécifique.
  • Les données renvoyées incluent les données des vidéos individuelles ainsi que les valeurs cumulatives ou moyennes de la liste de lecture
  • Si les données renvoyées n'incluent pas d'éléments pour toutes les vidéos de la playlist, aucune donnée n'a été renvoyée pour ces vidéos au cours de la période spécifiée

Logique de l'application

Il s'agit d'une application hybride qui utilise le CMS API pour récupérer les informations de la liste de lecture, puis le Analytics API pour récupérer des analyses pour les vidéos de la liste de lecture. Il y a une légère torsion: pour la liste de lecture EXPLICIT (manuelle), les métadonnées incluent les identifiants vidéo, nous pouvons donc les transférer directement vers Analytics API ; pour les listes de lecture intelligentes, cependant, nous devons revenir à CMS API en utilisant les critères de recherche, de tri et de limitation de la playlist pour obtenir les informations sur la vidéo, que nous pouvons ensuite transférer vers Analytics API:

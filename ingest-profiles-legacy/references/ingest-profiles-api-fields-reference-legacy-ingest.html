Champs de profil

Si un utilisateur extrait par programme un profil dans le but de mettre à jour ce profil à l'aide du contenu récupéré, l'utilisateur devra nettoyer le profil, en supprimant des éléments tels que "id " pour les rendus, mais pas le profil, et "version " doit être entièrement supprimé.
Biens Taper La description Défaut
id chaîne de caractères L'identifiant du profil (lecture seule) rien
version entier La version du profil (lecture seule) rien
account_id entier Les Nuage vidéo identifiant de compte rien
name chaîne de caractères Le nom du profil - doit être unique dans le compte rien
description chaîne de caractères La description du profil rien
date_created entier (horodatage d'époque (millisecondes) Date de création du profil (lecture seule) rien
date_last_modified entier (horodatage de l'époque en millisecondes) Date de la dernière modification du profil (lecture seule) rien
brightcove_standard booléen Lecture seulement - false pour tous les profils personnalisés false
digital_master objet Définit si et comment archiver un master numérique false
numRenditions entier Nombre de rendus dans le profil, y compris les rendus d'images (lecture seule) rien
renditions déployer Tableau d'objets de rendu []
packages déployer Gamme de packages DRM []

Remarques

  • [1-1] reproduire le comportement de l'ancien Nuage vidéo système d'ingestion utiliser une valeur de 10 - voir Sécurité du contenu (DRM et HLSe) pour plus de détails
  • [1-2] Une qualité supérieure signifie un débit binaire plus élevé et vice versa. Valeurs possibles:
    1. Hautement compressé. Qualité visuelle médiocre, mais petits fichiers
    2. Qualité acceptable
    3. Bonne qualité. Mieux que la plupart des vidéos Web
    4. Grande qualité. Semble excellent
    5. Presque sans perte. Gros fichiers. Non recommandé sauf si vous prévoyez d'encoder à nouveau cette sortie
  • [1-3] Vous ne devriez pas avoir à descendre en dessous de 16 ; 16 est presque sans perte pour la plupart des fichiers. Autour de 24 semble généralement assez bon. Environ 35 semble assez compressé
  • [1-4] Plus de trames de référence entraînent une qualité de compression légèrement supérieure, mais une complexité de décodage accrue. En pratique, aller au-dessus de 5 a rarement beaucoup d'avantages. La valeur par défaut est 3 comme un bon compromis de complexité de compression et de décodage. Réglez sur "auto" pour permettre à notre réglage de vitesse de choisir naturellement ce nombre
  • [1-5] Les niveaux inférieurs sont plus faciles à décoder, mais les niveaux supérieurs offrent une meilleure compression. Par exemple, l'iPhone 3gs ne prend en charge que le profil Baseline. Les profils Main et High constituent une nette amélioration de la compression et fonctionnent bien pour la lecture Web, mais soyez prudent avec la vidéo HD High Profile dans Flash Player
  • [1-6] Principalement utilisé pour la compatibilité des appareils. Par exemple, l'iPhone prend en charge le H.264 niveau 3, ce qui signifie que le decoder_bitrate_cap d'une vidéo ne peut pas dépasser 10 000 kbps. En règle générale, vous ne devez modifier ce paramètre que si vous ciblez un appareil spécifique qui en a besoin.
  • [1-7] Les images B ne sont prises en charge que par H.264 Main ou High profile, pas par Baseline. L'utilisation d'images B peut augmenter la qualité au même débit, mais est plus exigeante pour les processeurs. Cela peut être problématique sur les anciens ordinateurs ou appareils, mais fonctionne généralement sans problème sur le matériel moderne. Video Cloud prend par défaut la valeur 0 pour une compatibilité maximale. Nous recommandons une valeur de 3 pour les améliorations de compression/qualité. Les valeurs supérieures à 5 ou 6 offrent rarement beaucoup d'avantages et augmentent considérablement le temps d'encodage
  • [1-8] Pour " tiret' sorties, les profils actuellement pris en charge sont :
    • live- Mettra le profil DASH sur "urn:mpeg:dash:profile:isoff-live:2011 " et utilisez un seul SegmentTemplate pour chaque AdaptationSet. Pour DRM, il utilisera un IV de 8 octets, et pour PlayReady utilisera la version 1.1, y compris un PSSH atome dans les médias
    • hbbtv_1.5- Mettra le profil DASH sur "urn:hbbtv:dash:profile:isoff-live:2012 " et incluez les attributs requis pour les représentations audio et vidéo. Pour DRM, il utilisera un IV de 16 octets, et pour PlayReady utilisera la version 1.0, à l'exclusion PSSH atomes
  • [1-9] Les filigranes doivent être ajoutés au niveau du rendu - à chaque rendu que vous souhaitez filigraner

Champs maîtres numériques

Les digital_master L'objet définit si et comment stocker un master numérique que vous pourrez utiliser pour le retranscodage ultérieur. Voir Archivage des masters pour les détails.

Biens Taper La description Défaut

distribute booléen Si le digital_master doit être poussé vers le CDN pour être disponible pour la lecture false

Champs du package

Les packages sont utilisés pour les DRM. Si votre compte n'est pas activé pour les DRM, laissez simplement le packages tableau vide.

Remarque : les packages font référence aux rendus par reference_id. Pour utiliser des packages, vos rendus doivent avoir reference_ids et les rendus doivent être définis dans le JSON du profil avant les paquets sont définis.

Biens Taper La description Défaut
drm déployer Type DRM (pour MPEG-DASH uniquement)[2-2] rien
package_type chaîne de caractères Type de package DRM[2-2] rien
renditions déployer Un tableau de chaînes, dont chacune est le reference_id pour le rendu dans le profil[2-3] []

Remarques

  • [2-1] Valeur actuellement prise en charge pour le drm tableau sont marlin , playready , et widevine
  • [2-2] Voir Sécurité du contenu (DRM et HLSe) pour les détails
  • [2-3] Notez que les rendus dans le profil doivent être définis dans le JSON avant les packages, afin que les reference_ids existent

Champs de filigrane

Les images en filigrane sont gravées dans chaque image vidéo dans le coin inférieur droit.

Biens Taper La description Défaut
url chaîne de caractères URL de l'image rien
width chaîne de caractères[3-2] largeur du filigrane en pixels ou en pourcentage de la largeur du cadre - par exemple 20%[3-1] rien
height chaîne de caractères[3-2] hauteur du filigrane en pixels ou en pourcentage de la hauteur du cadre - par exemple 10%[3-1] rien
x chaîne de caractères[3-2] distance entre le bord gauche de l'image vidéo et le centre du filigrane en pixels ou en pourcentage de la largeur de l'image rien
y chaîne de caractères[3-2] distance entre le bord supérieur de l'image vidéo et le centre du filigrane en pixels ou en pourcentage de la largeur de l'image rien

Remarques

  • [3-1] Si vous spécifiez uniquement la largeur, le rapport hauteur/largeur de l'image sera maintenu automatiquement
  • [3-2] Bien que le width , height , x , et y les champs sont des valeurs de chaîne, ils contiennent des nombres (pixels ou pourcentage) qui doit être des entiers - les valeurs décimales comme "5,2%" seront rejetées.

Champs de profil

Si un utilisateur extrait par programme un profil dans l'intention de l'utiliser comme base pour un nouveau profil personnalisé, l'utilisateur devra supprimer le identifiant , version , date créée , date_dernière_modification , et Brightcove_standard champs, et le Nom doit être nouveau et unique dans le compte.

Champs d'options de profil dynamique

Les champs d'options de profil dynamique fournissent des instructions pour la création de rendu lorsque vous utilisez le codage contextuel.

Options de profil dynamique
Biens Taper La description JJ CAE Défaut
video_codecs Déployer Pour les sorties de codecs mixtes, un tableau des codecs pour lesquels créer des rendus. Actuellement, les valeurs d'élément autorisées sont h264 et hevc [2-1] rien
video_codec_options Déployer Pour les sorties de codec mixtes, un objet spécifiant les options pour chaque codec. Chaque propriété est un objet portant le nom h264 ou hevc , ses propriétés sont l'ensemble des contraintes pour les rendus pour ce codec. Les propriétés autorisées pour ces objets sont :
  • min_renditions obligatoire
  • min_bitrate
  • max_bitrate
  • max_first_rendition_bitrate
  • min_resolution
  • max_resolution
  • max_first_rendition_resolution
Remarque : ces paramètres sont également utilisés au niveau global. Lorsque vous utilisez des paramètres aux niveaux global et par codec, les valeurs doivent être compatibles. Par exemple, max_renditions défini au niveau global doit être supérieur ou égal à la somme des valeurs min_rendition définies au niveau par codec
 [2-2] rien
min_renditions Nombre Le nombre minimum de rendus à créer (obligatoire) rien
max_renditions Nombre Le nombre maximal de rendus à créer (obligatoire) rien
min_resolution objet La résolution minimale pour les rendus rien
min_resolution.width numéro La largeur minimale des rendus en pixels rien
min_resolution.height numéro La hauteur minimale des rendus en pixels rien
max_resolution objet La résolution maximale pour les rendus par défaut la résolution source
max_resolution.width numéro La largeur maximale des rendus en pixels largeur de la source
max_resolution.height numéro La hauteur maximale des rendus en pixels hauteur de la source
max_bitrate numéro Le débit binaire maximal pour les rendus en kbps débit source
max_first_rendition_bitrate numéro Le débit binaire maximal pour le rendu du débit binaire le plus bas en kbps débit source
max_frame_rate numéro La fréquence d'images maximale pour les rendus en fps 30
keyframe_rate numéro Le nombre d'images clés par seconde pour les rendus (donc une valeur de 0,5 entraînerait une image clé toutes les deux secondes ; une valeur de 3 entraînerait trois images clés par seconde) 0,5
select_baseline_profile_configuration booléen Au moins un rendu utilisé dans le profil sera le profil de base faux
select_distinct_configurations booléen Si vrai, CAE s'assurera que deux rendus n'utilisent pas la même combinaison de : résolution, fréquence d'images et profil de codec faux
watermarks déployer Tableau de filigrane objets spécifiant un ou plusieurs filigranes à ajouter aux rendus faux

Remarques

  • [2-1] Obligatoire si video_codec_options est présent. Voir Vidéo HEVC pour plus d'informations.
  • [2-2] Obligatoire si video_codecs est présent. Voir Vidéo HEVC pour plus d'informations.

    • Champs de filigrane

    Les images en filigrane sont gravées dans chaque image vidéo dans le coin inférieur droit.

    Notez que les filigranes peuvent être spécifiés dans les profils CAE via les champs indiqués ci-dessous. Si vous souhaitez ajouter des filigranes à un profil d'acquisition de livraison dynamique statique, contactez l'assistance Brightcove.
    Biens Taper La description Défaut
    url chaîne de caractères URL de l'image rien
    width chaîne de caractères[3-2] largeur du filigrane en pixels ou en pourcentage de la largeur du cadre - par exemple 20%[3-1] rien
    height chaîne de caractères[3-2] hauteur du filigrane en pixels ou en pourcentage de la hauteur du cadre - par exemple 10%[3-1] rien
    x chaîne de caractères[3-2] distance entre le bord gauche de l'image vidéo et le centre du filigrane en pixels ou en pourcentage de la largeur de l'image rien
    y chaîne de caractères[3-2] distance entre le bord supérieur de l'image vidéo et le centre du filigrane en pixels ou en pourcentage de la largeur de l'image rien

    Remarques

    • [3-1] Si vous spécifiez uniquement la largeur, le rapport hauteur/largeur de l'image sera maintenu automatiquement
    • [3-2] Bien que le width , height , x , et y les champs sont des valeurs de chaîne ; ils contiennent des nombres (pixels ou pourcentage)

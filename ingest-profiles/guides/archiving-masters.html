introduction

Les informations de cette rubrique s'appliquent principalement aux profils d'acquisition personnalisés. Profils standard tous les maîtres d'archives (voir au dessous de)

Si vous archivez un master lorsque vous ingérez une vidéo, vous pourrez re-transcoder la vidéo plus tard sans l'ingérer à nouveau. Cette rubrique explique vos options pour l'archivage de vos masters. Notez que l'archivage du master signifie stocker le fichier vidéo master dans votre compte afin qu'il soit disponible pour un traitement futur. Voir Création de profils d'ingestion pour la livraison dynamique pour plus d'informations.

Notez que si vous stockez le master numérique lorsque vous ingérez la vidéo, puis retranscodez la vidéo à l'aide d'un profil qui n'archive pas les masters, le master numérique de la vidéo sera ne pas être supprimé et vous pourrez toujours retranscoder la vidéo.

Archivage d'un master

Pour archiver un master pour votre vidéo, vous devez inclure le digital_master objet dans votre profil d'ingestion:

    {
        "account_id": 3423487123048,
        "digital_master": {
            "rendition": "passthrough",
            "distribute": false
        },
        "name": "nondrm-default",
        ...
    }

Source ou rendu

Pour les profils de livraison dynamique (y compris l'encodage contextuel), vous pouvez archiver le fichier source vidéo en tant que maître (en d'autres termes, définir "rendition": "passthrough" dans le digital_master partie du profil). Le digital_master va ne pas être distribué (poussé vers le CDN en tant que rendu disponible) pour ces profils, même si vous définissez "distribute": true.

Pour les profils Legacy Ingest, vous pouvez choisir d'utiliser votre fichier source vidéo comme maître ou vous pouvez sélectionner un rendu de haute qualité à utiliser comme maître.

Pour utiliser le fichier source vidéo que vous ingérez en tant que maître, incluez le code suivant dans votre profil d'acquisition:

    "digital_master": {
        "rendition": "passthrough",
        "distribute": false
    }

Pour utiliser un rendu pour le maître au lieu de votre fichier source (profils Legacy Ingest uniquement), définissez le rendition domaine de la digital_master s'opposer à la Pièce d'identité pour le rendu :

    "digital_master": {
            "rendition": "av0",
            "distribute": true
        },
        "name": "nondrm-av0",
        "packages": [],
        "renditions": [
        {
            "media_type": "video",
            "reference_id": "av0", ...

Si vous définissez rendition à none , aucun master ne sera archivé , et tu vas ne pas pouvoir retranscoder la vidéo. Vous pouvez cependant remplacer la vidéo en l'intégrant à nouveau.

Distribution

La distribution ne fonctionnera que pour Profils d'intégration hérités.

Vous pouvez choisir si le maître est distribué ou non. Distribué signifie que le master est poussé vers le CDN et disponible en tant que rendu jouable. Si le master n'est pas distribué, il est archivé par Video Cloud mais pas poussé vers le CDN.

Remarques:
  • Si vous avez sélectionné l'un de vos rendus comme maître et défini distribute à false , ce rendu ne sera pas transmis au CDN et ne sera pas disponible pour la lecture
  • Si le rendu est défini sur passthrough et distribute à true , le master ne sera distribué que s'il est au format h264
  • La vidéo originale peut être modifiée afin de la rendre jouable sur Internet. Le master archivé sera cependant identique au fichier d'origine, sauf dans de très rares cas où nous devons le modifier afin de le transcoder
  • La distribution du maître fait ne pas travailler pour les vidéos ingérées en utilisant Livraison dynamique profils. Si vous regardez les profils d'ingestion de Dynamic Delivery, vous verrez que distribute est réglé sur false. Si vous dupliquez le profil et tentez de le modifier dans Studio, la valeur reviendra à false automatiquement.

Pour distribuer le maître au CDN, définissez "distribute": true dans le digital_master objet:

    "digital_master": {
        "distribute": true,
        "rendition": "passthrough"
    }

Pour empêcher la distribution du maître, définissez distribute à faux:

    "digital_master": {
        "distribute": false,
        "rendition": "passthrough"
    }

Comportement par défaut

Par défaut (si le profil ne comprend pas de digital_master champ), le Dynamic Ingest API et Module de téléchargement se comportera exactement comme si les éléments suivants avaient été inclus dans le profil:

    "digital_master": {
        "distribute": false,
        "rendition": "none"
    }

En d'autres termes, aucun master numérique ne sera stocké.

Si vous avez les éléments suivants dans votre profil:

    "digital_master": {
        "distribute": false
    }

... sans rendition champ pour spécifier ce qui doit être archivé, la vidéo source sera archivé en tant que maître numérique.

Profils standards

Les profils standards pour Livraison dynamique ainsi que le système d'ingestion hérité stockera automatiquement les masters mais ne pas les distribuer :

Standard Livraison dynamique profils

  • multi-plateforme-étendu-statique

  • multi-plateforme-standard-statique
  • multi-plateforme-standard-statique-avec-MP4
  • multi-plateforme-étendu-dynamique
  • multi-plateforme-étendu-dynamique-avec-MP4
  • multi-plateforme-standard-dynamique
  • multi-plateforme-standard-dynamique-avec-MP4
  • dynamique à faible bande passante

    Implémentation des règles

    Dans cette rubrique, vous apprendrez comment mettre en œuvre les règles de livraison de Brightcove, qui vous permettent de personnaliser la façon dont vos médias sont diffusés afin d'atteindre vos objectifs commerciaux.
    Cette fonctionnalité est disponible pour un ensemble spécifique de clients ayant accès à la phase de disponibilité limitée des Règles de livraison.

    Présentation

    Les éditeurs doivent souvent modifier les caractéristiques de contenu de leurs manifestes médias pour des raisons techniques et commerciales. Pour répondre à ces besoins, Brightcove a créé un cadre dans lequel des règles peuvent être appliquées qui modifieront le contenu du manifeste renvoyé par Dynamic Delivery.

    Les règles de livraison sont constituées de conditions et d'actions.

    • Conditions - Conditions « Si » qui déclenchent une certaine action
    • Actions - Paramètres « Alors » qui définissent la façon dont le manifeste est modifié

    Pour plus d'informations sur les règles de livraison, consultez les sections suivantes :

    • Vue d'ensemble : Règles de livraison
    • Utilisation des règles de livraison avec Brightcove Player
    • Utilisation des règles de mise à disposition avec les kits SDK natifs
    • Vue d'ensemble de la livraison dynamique

    Configuration du compte

    Contactez votre gestionnaire de compte pour en savoir plus sur la tarification et activer cette fonctionnalité pour votre compte.

    Application des règles

    Le diagramme de flux suivant montre comment les règles de remise sont appliquées.

    Conditions

    Les conditions « si » prises en charge dans la première phase sont les suivantes :

    • Demande explicite - Un ID de configuration est l'identifiant d'une action que vous avez créée. Vous utiliserez cet identifiant pour passer à l'API de lecture lors de l'exécution. Cet identifiant est mappé aux valeurs de configuration qui permettent aux services sous-jacents de prendre des décisions exploitées par l'API Playback pour renvoyer les données de réponse appropriées.

    • Groupe de périphériques : type de périphérique détecté par l'analyse de l'agent utilisateur. Les groupes actuellement pris en charge sont les suivants :
      • android
      • appletv
      • iPad
      • iphone
      • ordinateur de bureau chromé
      • bureau-firefox
      • bureau-safari
      • fetchtv
      • ios-autre
      • roku-7/8
      • smartphone-générique
      • tablette-android
      • tablette-Kindle
      • inconnu
       
    • Géographie - Emplacement physique de l'appareil requérant :
      • Continent
      • Pays

    Actions

    Les actions, ou paramètres « puis », prises en charge dans la première phase comprennent les éléments suivants :

    • Media Delivery CDN - À partir des CDN configurés pour être utilisés dans un compte Delivery dynamique donné, le CDN à utiliser pour la livraison de segments de médias.

    • Caractéristiques du rendu - Filtres basés sur les caractéristiques des formats associés multimédia sous-jacents. Ils nous informent ainsi que :
      • Débit vidéo minimum
      • Débit vidéo maximal
      • Résolution vidéo minimale
      • Résolution vidéo maximale
      • Nombre maximal de formats associés audio
      • Nombre maximal de formats associés vidéo
      • Premier débit vidéo
      • Discontinuités

    Portée

    Les actions peuvent être appliquées à deux niveaux :

    • Compte - Actions qui s'appliquent à toutes les demandes adressées au compte d'un client spécifique
    • Demande spécifique - Actions qui sont invoquées sur une demande spécifique

    Les actions seront appliquées dans l'ordre indiqué ci-dessus. Les actions ultérieures remplaceront les précédentes.

    Définir des règles

    Utilisez l'API Règles de livraison pour personnaliser la remise des médias.

    API des règles de livraison

    L' API Règles de livraison vous permet de définir des conditions et des actions pour contrôler votre diffusion multimédia.

    URL de base

    L'URL de base de l'API est : 

    https://delivery-rules.api.brightcove.com

    Chemin du compte

    Dans tous les cas, des demandes seront faites pour un compte Video Cloud spécifique. Ainsi, vous ajouterez toujours les comptes de terme suivis de votre identifiant de compte à l'URL de base : 

    https://delivery-rules.api.brightcove.com/accounts/{accountID}

    Autorisation

    Un jeton d'accès pour les requêtes est requis et doit être présent dans l'en-tête Autorisation። 

    Authorization: Bearer {access_token}

    Le jeton d'accès est un jeton d'accès OAuth2 temporaire qui doit être obtenu à partir du service Brightcove OAuth. Pour plus d'informations sur la façon d'obtenir des informations d'identification client et de les utiliser pour récupérer des jetons d'accès, consultez la présentation OAuth de Brightcove.

    Autorisations

    Les demandes à l'API Droits de lecture doivent être faites à partir des informations d'identification du client avec les autorisations suivantes :

    • video-cloud/delivery-rules/read
    • video-cloud/delivery-rules/all

    Gérer les règles

    L'API Règles de livraison prend en charge les demandes suivantes. Pour plus d'informations sur l'API, consultez la référence de l'API des règles de livraison.

    Obtenir les règles de livraison

    Utilisez la requête GET suivante pour récupérer la dernière version des règles de livraison pour un compte. 

    GET /accounts/{accountID}
    Réponse de l'API

    Voici le corps de réponse. Notez que les règles de remise contiennent un ensemble de conditions et un ensemble d'actions. 

    {
  "conditions": [
    {
      "name": "string",
      "if": {
        "device_group": [
          "ipad"
        ],
        "request_country": [
          "string"
        ],
        "request_continent": [
          "AF"
        ]
      },
      "then": [
        "string"
      ]
    }
  ],
  "actions": [
    {
      "id": "string",
      "properties": {
        "min_video_bitrate": 0,
        "max_video_bitrate": 0,
        "first_video_bitrate": 0,
        "min_video_resolution": "string",
        "max_video_resolution": "string",
        "max_video_renditions": 0,
        "max_audio_renditions": 0,
        "preferred_cdn_provider": "string",
        "preferred_cdn_domain": "string",
      "video_codecs": [
        "string"
      ],
      "audio_codecs": [
        "string"
      ]
    }
    }
  ]
}

    Obtenir les conditions

    Utilisez la requête GET suivante pour récupérer les conditions d'un compte. 

    GET /accounts/{accountID}/conditions
    Réponse de l'API

    Voici le corps de réponse. 

    [
  {
    "name": "string",
    "if": {
      "device_group": [
        "ipad"
      ],
      "request_country": [
        "string"
      ],
      "request_continent": [
        "AF"
      ]
    },
    "then": [
      "string"
    ]
  }
]

    Mises à jour

    Utilisez la requête PUT suivante pour mettre à jour les conditions d'un compte. 

    PUT /accounts/{accountID}/conditions
  Content-Type: application/json
  Authorization: Bearer {access_token}
  Body: {conditions object}

    Organisme de demande pour les conditions

    Voici le corps de requête pour les conditions. 

    {
  "name": "string",
  "if": {
    "device_group": [
      "ipad"
    ],
    "request_country": [
      "string"
    ],
    "request_continent": [
      "AF"
    ]
  },
  "then": [
    "string"
  ]
}

    Voici les détails du champ pour les conditions :

    Champ Type Description
    name Chaîne Identifiant unique pour la condition
    device_group Chaîne Tableau de types de périphériques détectés par l'analyse de l'agent utilisateur
    request_country Chaîne Tableau de codes pays à deux lettres
    request_continent Chaîne Tableau de codes continents à deux lettres
    then Chaîne Identifiant unique pour l'action associée

    Créer une action

    Utilisez la requête POST suivante pour créer des actions pour un compte. 

    POST /accounts/{accountID}/actions
  Content-Type: application/json
  Authorization: Bearer {access_token}
  Body: {actions object}

    Corps de demande d'actions

    Voici le corps de la requête pour les actions. 

    {
    "properties": {
      "min_video_bitrate": 0,
      "max_video_bitrate": 0,
      "first_video_bitrate": 0,
      "min_video_resolution": "string",
      "max_video_resolution": "string",
      "max_video_renditions": 0,
      "max_audio_renditions": 0,
      "preferred_cdn_provider": "string",
      "preferred_cdn_domain": "string",
      "video_codecs": [
        "string"
      ],
      "audio_codecs": [
        "string"
      ]
  }
}

    Réponse de l'API pour les actions

    Voici le corps de la requête pour les actions. 

    {
    "id": "string",
    "properties": {
      "min_video_bitrate": 0,
      "max_video_bitrate": 0,
      "first_video_bitrate": 0,
      "min_video_resolution": "string",
      "max_video_resolution": "string",
      "max_video_renditions": 0,
      "max_audio_renditions": 0,
      "preferred_cdn_provider": "string",
      "preferred_cdn_domain": "string",
      "video_codecs": [
       "string"
      ],
      "audio_codecs": [
       "string"
      ]
  }
}

    Voici les détails des champs pour les actions :

    Champ Type Description
    id Chaîne Identifiant unique généré par le système pour l'action. Ceci est actionID utilisé dans les méthodes de mise à jour et de suppression.
    properties Objet Objet de propriétés définies pour une action spécifique
    min_video_bitrate,
    max_video_bitrate    		 Entier Définir le débit vidéo minimum ou maximal autorisé
    first_video_bitrate Entier Réglez le débit de la première vidéo
    min_video_resolution,
    max_video_resolution    		 Chaîne Définir la résolution vidéo minimale ou maximale autorisée
    max_video_renditions Entier Définir le nombre maximal de formats associés vidéo
    max_audio_renditions Entier Définir le nombre maximal de formats associés audio
    preferred_cdn_provider Chaîne Définir le fournisseur CDN préféré
    preferred_cdn_domain Chaîne Définir le domaine CDN préféré
    video_codecs Chaîne Tableau de codecs vidéo
    audio_codecs Chaîne Tableau de codecs audio

    Obtenir des actions

    Utilisez la requête GET suivante pour récupérer les actions d'un compte. 

    GET /accounts/{accountID}/actions
    Réponse de l'API

    Voir la réponse de l'API pour les actions.

    Obtenir une action

    Utilisez la requête GET suivante pour récupérer une action spécifique pour un compte. 

    GET /accounts/{accountID}/actions/{actionID}
    Réponse de l'API

    Mettre à jour une action

    Utilisez la requête PUT suivante pour mettre à jour une action pour un compte. 

    PUT /accounts/{accountID}/actions/{actionID}
  Content-Type: application/json
  Authorization: Bearer {access_token}
  Body: {actions object}

    Supprimer une action

    Utilisez la demande DELETE suivante pour supprimer une action pour un compte. 

    DELETE /accounts/{accountID}/actions/{actionID}

    Contraintes

    Il existe quelques limitations connues lors de l'utilisation des règles de livraison :

    • Contenu multimédia - Les règles de diffusion ne fonctionneront pas avec les vidéos Smooth ou PMP4.
    • Audio uniquement - Les règles de diffusion avec les annonces côté serveur (SSAI) activées nécessitent à la fois du contenu vidéo et audio. Il s'agit d'une restriction SSAI.

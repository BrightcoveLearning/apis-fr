introduction

Les règles de livraison de Brightcove sont une fonctionnalité de la livraison dynamique qui vous permet de tirer parti de la capacité de génération de manifestes juste à temps sur une base plus granulaire. Avec les règles de livraison, vous pouvez créer des règles personnalisées pour contrôler la manière dont votre contenu est livré au spectateur. Cela vous permet de diffuser du contenu de manière plus efficace et efficiente.

Les règles de livraison vous donnent la possibilité de créer une logique conditionnelle basée sur le type d'appareil, l'emplacement de l'utilisateur ou un signal de votre application. Lorsque ces conditions sont déclenchées, vous contrôlez le comportement tel que les types de rendus renvoyés et le CDN à utiliser.

Pour plus d'informations sur la mise en œuvre des règles de livraison, consultez les éléments suivants :

Exemples

Comme vous pouvez l'imaginer, les combinaisons de logique conditionnelle peuvent être infinies. Voici quelques exemples de ce que les règles de livraison peuvent faire :

Plusieurs niveaux d'abonnement

Un modèle courant pour les services de vidéo à la demande par abonnement (SVOD) consiste à proposer deux niveaux d'abonnement : définition standard (SD) et haute définition (HD).

Sur la base d'un signal de votre application, vous pouvez indiquer à Dynamic Delivery de renvoyer uniquement les rendus SD pour les utilisateurs standard et les rendus HD pour les utilisateurs premium.

CDN optimisés pour la géographie

Alors que la plupart des CDN sont optimisés pour une distribution mondiale, certaines zones sont mieux desservies par un CDN spécifique à une région. Un CDN dans la région ou dans le pays peut être utilisé pour une livraison optimale.

Avec les règles de livraison, vous pouvez créer une règle de sorte que lorsqu'un spectateur se trouve dans une zone spécifique, un CDN spécifique à la région sera utilisé. Pour tous les autres utilisateurs, un CDN global sera utilisé.

Rendus optimisés pour les appareils

Étant donné que les consommateurs visualisent les médias à partir d'une variété de tailles d'écran, il n'est pas idéal de fournir le même ensemble de rendus à chaque appareil.

Avec les règles de livraison, vous pouvez créer une règle qui détecte si l'appareil du spectateur est un téléphone et exclure les rendus 1080p. Vous pourriez avoir une autre règle qui détecte si un spectateur se trouve sur un appareil OTT et exclut les rendus inférieurs à 360p.
